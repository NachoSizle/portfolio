## Sobre el Proyecto

Este portfolio está planteado como un producto profesional, no solo como una web de presentación. El objetivo es comunicar mi perfil técnico con claridad, cargar rápido y mantenerse fácil de evolucionar.

Lo diseñé para que la experiencia de quien lo visita (recruiter, manager o equipo técnico) sea directa: entender quién soy, qué construyo y cómo trabajo.

## Características Principales

### Qué problema resolvía

- Presentar marca personal y capacidad técnica sin ruido ni fricción.
- Evitar un portfolio centrado en "stack" y llevarlo hacia una narrativa más útil para contratación.
- Mantener una base que pueda evolucionar rápido con nuevo contenido.

### Qué decisiones tomé

- **Performance como requisito de producto**: páginas estáticas, assets optimizados y carga rápida en móvil.
- **SEO y accesibilidad como estándar**: no como checklist de cierre.
- **Bilingüe desde arquitectura**: contenido y navegación preparados para ES/EN.
- **DX y mantenibilidad**: estructura clara para iterar narrativa y proyectos sin romper el sitio.
- **Métricas como referencia**: Lighthouse y Core Web Vitals como señal objetiva de calidad técnica.

### Qué construí

- Sitio en Astro con generación estática y estructura modular.
- Sistema i18n con rutas y copy en español e inglés.
- Base SEO (meta, canonical, sitemap) y buenas prácticas de accesibilidad.
- Secciones de home y proyectos enfocadas en comunicar perfil técnico y criterio de producto.
- Pipeline de build/deploy preparado para publicación continua.
- Pipeline con GitHub Actions y Bun para checks, test y build en cada iteración.

## Impacto y aprendizaje

- Validé que el diseño y el contenido también son decisiones de producto.
- Confirmé que una arquitectura limpia mejora la velocidad de ejecución y la claridad del mensaje.
- Aprendí a tratar el portfolio como un producto vivo, con iteración continua.

## Stack (soporte)

- Astro, Tailwind CSS, TypeScript y Bun.
- i18n, optimización de imágenes y pipeline de despliegue en GitHub Pages.
